Manifest Data Model — Current vs Proposed

How do we represent a robot's physical parts?

Current Model (broken)

Two parallel lists that overlap
bodies[] — kinematic chain
{ name: "base", mesh: "base.stl", kind: "fabricated" }
{ name: "left_rim", mesh: "left_rim.stl", kind: "fabricated" }
{ name: "right_rim", mesh: "right_rim.stl", kind: "fabricated" }
⚠ "left_rim" is really the Pololu wheel — not a separate rim
parts[] — things on bodies
{ id: "comp_base_battery", parent_body: "base", mesh: "comp_base_battery.stl" }
{ id: "comp_base_camera", parent_body: "base", mesh: "comp_base_camera.stl" }
{ id: "comp_base_pi", parent_body: "base", mesh: "comp_base_pi.stl" }
{ id: "comp_left_rim_wheel", parent_body: "left_rim", mesh: "comp_left_rim_wheel.stl" }
{ id: "comp_right_rim_wheel", parent_body: "right_rim", mesh: "comp_right_rim_wheel.stl" }
{ id: "servo_left_wheel", ... }
{ id: "horn_left_wheel", ... }
fasteners...
⚠ comp_left_rim_wheel.stl is IDENTICAL to left_rim.stl — same bytes
Problem: One physical object (the wheel) appears as BOTH a body AND a part. The viewer loads both = double render. The body list is really "kinematic nodes for MuJoCo" not "physical parts to render."

Proposed Model

One list of bodies — each is a real physical part
bodies[] — every rigid body, with its role
{ name: "base", mesh: "base.stl",
role: "structure", shape: "tube",
mounts: ["battery", "camera", "pi"] }
{ name: "left_rim", mesh: "left_rim.stl",
role: "component", component: "Pololu 90x10mm Wheel" }
{ name: "right_rim", mesh: "right_rim.stl",
role: "component", component: "Pololu 90x10mm Wheel" }
{ name: "servo_left_wheel", mesh: "servo_STS3215.stl",
role: "component", component: "STS3215" }
{ name: "horn_left_wheel", mesh: "horn_left_wheel.stl",
role: "component", component: "Horn disc" }
... (same for right side)
mounts[] — components mounted ON structural bodies
{ body: "base", label: "battery", component: "LiPo2S-1000",
mesh: "comp_base_battery.stl", pos: [...], quat: [...] }
{ body: "base", label: "camera", component: "OV5647",
mesh: "comp_base_camera.stl", pos: [...], quat: [...] }
{ body: "base", label: "pi", component: "RaspberryPiZero2W",
mesh: "comp_base_pi.stl", pos: [...], quat: [...] }
✓ Only parts mounted ON a structural body — not bodies that ARE components
joints[] — connections between bodies
{ name: "left_wheel", parent: "base", child: "left_rim", axis: [...] }
{ name: "right_wheel", parent: "base", child: "right_rim", axis: [...] }
Key change: Servos, horns, and wheels are bodies (they're rigid things in the kinematic chain). Battery, camera, Pi are mounts (attached to a structural body, not their own kinematic node). No duplication.
